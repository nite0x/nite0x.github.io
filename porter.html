// how it works

Zero config.
Install once, forget about it.

01 —
🔌

Install the daemon

A lightweight Go process runs in the background, watching your system's network activity.

$ brew install porter-app
$ porter daemon start
✓ daemon running
02 —
🔍

Auto-discovery kicks in

Porter detects every listening port, maps it to its process, and figures out what's running — automatically.

scanning ports...
✓ Vite :1420
✓ PostgreSQL :5432
✓ MCP Inspector :6274
03 —
🎯

Open, manage, control

One click to open any service in the browser. Ignore noisy system services. Name things properly.

porter open 3000
porter ignore 7000
→ opening localhost:3000
// features

Built for developers
who run a lot of stuff.

Auto-discovery

Sees everything running on your machine

Porter scans all listening ports in real-time, identifies the process behind each one, and gives them friendly names. No Procfiles. No manual registration.

Vite:1420
PostgreSQL:5432
Redis:6379
ControlCenterignored
Project-aware

Groups services by project automatically

Services launched from the same directory are grouped together. Your frontend, backend, and database show up as one coherent project — not a random list of ports.

~/projects/myapp
Next.js:3000
API server:8080
PostgreSQL:5432
MCP server

Let Claude see your dev environment

Porter ships with a built-in MCP server. Connect it to Claude and ask natural questions about what's running — or let AI help you debug, restart, or manage services.

U
What services are currently running on my machine?
C
I can see 9 services running. Your main project looks like it's on port 3000 (Next.js) with an API server on 8080 and PostgreSQL on 5432. There's also an MCP Inspector on 6274 and 6277.
U
Which ones have been running longest?
C
PostgreSQL has been up for 3 days. The Next.js dev server started 2 hours ago.
